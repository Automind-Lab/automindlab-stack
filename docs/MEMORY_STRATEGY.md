# Runtime Memory Strategy

AutoMindLab should keep memory small, structured, and reviewable.

## Memory layers
- **incident memory** — reusable lessons from real issues
- **operator memory** — recurring operator preferences or constraints
- **workflow memory** — validated handoff or routing patterns
- **diagnostic memory** — recurring field patterns that improve consultation quality

## Rules
- only capture memory from validated outcomes or reviewed guidance
- keep entries short enough to rehydrate quickly
- never store secrets or private credentials
- attach source references whenever possible

## Suggested lifecycle
1. produce advisory output
2. confirm outcome or review it with a human
3. capture a short memory record
4. use that memory only as supporting context, not as proof
